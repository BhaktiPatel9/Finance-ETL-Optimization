# Financial ETL & Reconciliation Optimization

**Business Impact:** Reduced monthly financial closing cycles by 8+ hours and cut manual reconciliation errors by 40%.

## Project Overview
At K.G. Patel & Co., the monthly financial closing process was bottlenecked by slow-running SQL queries and manual data reconciliation across multi-terabyte datasets. This project focuses on the database optimization and automated ETL logic required to scale the financial reporting pipeline.

## Tech Stack
* **Database:** SQL Server
* **ETL:** Azure Data Factory
* **Visualization:** Power BI / Tableau
* **Techniques:** Query Optimization, Indexing, Common Table Expressions (CTEs), Window Functions

## Key Deliverables
1. **Performance Tuning:** Refactored legacy nested subqueries into optimized CTEs, improving execution time by 25%.
2. **Automated Reconciliation:** Replaced manual Excel cross-checks with automated SQL validation scripts.
3. **Data Governance:** Ensured inventory valuation accuracy improved by 20% through standardized data types and constraints.
